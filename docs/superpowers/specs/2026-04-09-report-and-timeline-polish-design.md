# 报告页与时间线体验修复设计

**目标**

修复报告页生成态遮挡、英文长模型名遮挡、AI 增强配置与已保存日报生成方式语义混淆、时间线详情图点击后延迟明显，以及时段摘要信息层过薄的问题。

**范围**

- 报告页 `Report.svelte` 的页头、昨日报告提示条、生成态布局与元信息展示
- 报告页元信息解析逻辑抽离为独立 helper，便于测试“当前配置”和“当前日报实际生成方式”
- 时间线详情打开流程优化为并行请求，并优先展示已有缩略图
- 时段摘要页在现有数据结构上增加副摘要与节奏信息，不新增后端接口

**设计决策**

1. 报告页头同时表达两层信息：
   - 主展示：当前这份日报实际生成方式
   - 次展示：当前配置的日报模式与模型

2. 当今天尚无日报、正在显示昨日报告时：
   - 保留“显示的是昨日报告”的提示
   - 生成按钮在窄宽度下改成整块自适应，不再压缩变形

3. 时间线详情点击后：
   - 先立刻打开弹窗
   - 立即用列表缩略图占位
   - `get_activity` 与高清图并行请求，减少串行等待

4. 时段摘要保留现有带状结构，但增加：
   - 一句副摘要
   - 时段节奏/强度胶囊
   - 关键应用数量或主应用组信息

**验收标准**

- 报告页在中文和英文模型名下均不发生标题区遮挡
- 昨日报告提示条中的生成按钮在加载态不再出现压扁/裁切
- 当配置为 AI 增强但当前日报实际为模板回退时，页面能同时表达这两个事实
- 时间线详情点击后弹窗立即出现，且优先显示已有缩略图
- 时段摘要相比当前版本增加至少一层辅助信息，视觉与信息密度更完整
